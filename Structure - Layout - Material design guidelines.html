UI 区域

有关在z轴中排列元素以获得正确行为和阴影的指导，请参阅“环境和高程”和“阴影”部分。

移动端构造

此结构包括一个永久的应用程序栏和浮动操作按钮。 可以添加可选的底部条，用于附加功能或动作溢出。 侧导航菜单覆盖所有其他结构元素。

平板的构造

此结构显示具有浮动操作按钮的永久应用程序栏。 应用程序栏吸收了平板电脑和移动底部的元素。 可以添加可选的底部条，用于附加功能或动作溢出。 侧面导航覆盖所有其他结构元素。 可以临时访问右导航菜单或固定永久显示。

桌面端结构

桌面结构包含一个带有浮动动作按钮的永久应用程序栏。 应用程序栏吸收了平板电脑和移动底部的元素。 在可能的情况下，窗口控件被吸收到应用程序栏中。

侧面导航菜单可以占据屏幕尺寸的全高（包括在应用程序栏下），并可临时访问或固定以进行永久显示。 侧导航菜单以及内容画布可以具有自己的辅助工具栏，用于选项卡，调色板或辅助操作。

UI 区域

定义主要的水平或垂直分隔线。

避免将界面切割成太多可能导致L形的区域。 相反，使用空白来描绘次要地区。

用卡片和浮动动作按钮打破边缘。

使用卡片来组织内容，当需要特定的行为或信息分组需要比空白或分隔符可以提供更多的分离。

工具栏

工具栏是多功能的，可以以多种不同的方式使用。

应用栏

应用程序栏，以前称为Android中的动作栏，是一种特殊的工具栏，用于品牌，导航，搜索和操作。

应用程序栏左侧的导航图标可以是：

  • 打开导航抽屉的控件。
  • 一个向上箭头，用于向上导航到您的应用程序的层次结构。
  • 如果不需要从此屏幕导航，则完全忽略。

应用程序栏中的标题反映了当前页面。 它可以是应用程序标题，页面标题或页面过滤器。

应用程序栏右侧的图标是与应用程序相关的操作。 菜单图标打开溢出菜单，其中包含辅助操作和菜单项，如帮助，设置和反馈。

标题颜色

在应用程序栏中，所有图标应该是相同的颜色。

如果需要增加视觉层次结构，标题可以具有与图标不同的颜色。 不同的标题颜色对于白色和黑色字形都具有足够的对比度的背景效果最好。

指标

指标：

Mobile Landscape: 48dp
Mobile Portrait: 56dp
Tablet/Desktop: 64dp

对于扩展应用程序栏，高度等于默认高度加内容增量。

密度

当鼠标和键盘是主要的输入方法时，尺寸可能会被压缩，以适应更加密集的布局。

菜单

菜单是一个临时的纸张，它始终与应用程序栏重叠，而不是作为应用程序栏的扩展。

系统栏

状态栏/窗口栏

在Android上，状态栏包含通知图标和系统图标。 在Chrome上，顶部栏包含窗口控件：最小化，全屏和关闭。 在Chrome应用程序中，顶部栏可以消失，然后将窗口控件带入应用程序栏。

指标:
Android status bar height: 24dp
Chrome window height: 32dp

全屏

一些内容在全屏模式下是最好的，例如视频，游戏，书籍和幻灯片。 全屏模式通过最大限度地减少控制中的视觉干扰并保护用户意外逃避应用，从而增加了对内容的参与度。

全屏模式有几种类型：

  • Lean back
  • 身临其境
  • Lights Out

Lean back

Lean back模式最适合用有限的屏幕交互来观看内容，例如视频。

互动: 在任何地方触摸屏幕以带回系统条。

沉侵式

沉浸式模式最适合用户需要与屏幕进行大量交互，例如玩游戏或与图像库交互。 您可以根据需要显示和隐藏您的应用程序的控件以及系统栏。

互动: 从屏幕的任何边缘滑动，使系统栏出现。 第一次应用程序全屏，这个滑动手势被解释。

边缘滑动异常： 使用滑动边缘手势执行动作的应用程序也应该在使用该手势时显示系统栏。 例如，使用边缘滑动（例如绘制线条）的绘图应用程序也应该在发生任何手势时半透明地显示系统条。

Lights Out

在Lights Out模式下，操作栏和状态栏会消失，几秒钟后就不可用。 导航栏仍然可用并响应触摸，但显示为灰色。

暗色状态栏

默认情况下，状态或窗口栏的颜色是应用程序栏颜色的阴影。 它也可以使用布局中另一个元素的颜色或半透明。

浅色状态栏

浅色状态栏，黑色图标，更好地与浅色内容协调一致，可以作为黑色状态栏的替代品。

Android导航栏

Android中的导航栏包含设备导航控件：“后退”，“主页”和“概述”。 它还显示为Android 2.3或更早版本编写的应用程序的菜单。

高度: 48dp

色彩变异

导航栏可以是不透明的，半透明的或透明的。

Chrome操作系统架

货架上安装了应用程序启动器，应用程序图标和Chrome操作系统上的系统设置。

高度: 56dp

侧边导航

如果存在，则可以将侧导杆固定为永久显示或临时浮动作为重叠。 侧面导航可以位于屏幕的左侧或右侧。

出现在左侧导航栏中的内容理想地是导航或基于身份的。 右侧的nav内容应该是页面上主要内容的次要功能。

另见 导航栏。

构造

侧导航栏可能被固定为永久显示，或者它们可以临时浮动作为叠加。 临时导航抽屉覆盖内容画布; 而固定导航面板位于内容画布旁边或下方。

屏幕尺寸可以确定面板是否被固定或覆盖。 有足够的空间，可以对面板进行固定和内容调整。 空间不足，面板必须是叠加层。

指标

移动端:

宽度=屏幕宽度 - 56 dp

最大宽度：320dp

最大宽度仅适用于使用左导航。 当使用右侧导航面板时，面板可以覆盖屏幕的全部宽度。

桌面: 左导航的最大宽度为400dp。 右侧的导航可以根据内容而有所不同。

白框

白框提供了各种布局结构，使用一致的方法来进行曲面，分层和阴影。

有关白框的下载和其他信息，请参阅参考资料。
